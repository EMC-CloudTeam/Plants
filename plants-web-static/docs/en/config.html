Configuring mail resources (optional)

Plants by WebSphere can be configured to send an order confirmation e-mail. A mail provider for the Plants by WebSphere Sample may have already been created during installation, and can be further configured. If you do not want to send an order confirmation e-mail, skip this section. Follow these steps to configure mail resources:


  1. In the Integrated Solutions Console, click Resources > Mail > Mail Providers > (set scope to Node <node>) > Built-in Mail Provider > Mail Sessions.
  2. Click New to create a new mail session, or if the PlantsByWebSphere Mail Session already exists, click to edit it.
  3. Enter PlantsByWebSphere mail session in the Name field.
  4. Enter mail/PlantsByWebSphere in the JNDI Name field.
  5. In the Outgoing Mail Properties section, enter a value, such as yourcompany.com, in the Server field.
  6. Optionally, in the Outgoing Mail Properties section, enter values in the User and Password fields if required by your server.
  7. In the Outgoing Mail Properties section, enter a value, such as userid@yourcompany.com in the Return e-mail address field.
  8. Click OK.
  9. Click Save at the top of the main panel in the administrative console.
  10. Restart WebSphere Application Server so that all configuration changes are applied.

Mail resources for the Plants by WebSphere Sample are now configured.
